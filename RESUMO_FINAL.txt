PASSO 1: Inicializar repositório Git
  cd "c:\Users\User\Downloads\Moderntaskmanagerinterface-main\Moderntaskmanagerinterface-main"
  git init
  git add .
  git commit -m "Initial commit: Task Manager - React 19 + FastAPI + PostgreSQL"

PASSO 2: Fazer push para GitHub (opcional mas recomendado)
  Consulte: GITHUB_PUSH.txt
  
  git remote add origin https://github.com/seu-usuario/task-manager.git
  git push -u origin main

PASSO 3: Instalar e rodar (em sequência)
  
  Terminal 1 (Docker - PostgreSQL):
    docker-compose up -d postgres
    (Aguarde 10 segundos para o banco estar pronto)
  
  Terminal 2 (Backend):
    cd backend
    python -m venv venv
    venv\Scripts\activate  (Windows) ou source venv/bin/activate (Mac/Linux)
    pip install -r requirements.txt
    python run.py
    
    Você deve ver: Uvicorn running on http://127.0.0.1:8000
  
  Terminal 3 (Frontend):
    npm install
    npm run dev
    
    Você deve ver: Local: http://localhost:5173

PASSO 4: Testar
  - Abra http://localhost:5173 no navegador
  - Backend Swagger: http://localhost:8000/docs
  - Adminer (opcional): http://localhost:8080

================================================================================
  💡 DICAS IMPORTANTES
================================================================================

1. Sempre tenha 3 terminais abertos:
   - Terminal 1: Docker (PostgreSQL)
   - Terminal 2: Backend (FastAPI)
   - Terminal 3: Frontend (Vite)

2. Cheque se o banco está pronto antes de rodar o backend:
   docker ps (deve mostrar task_manager_db como "Up")

3. Use o Swagger para testar a API:
   http://localhost:8000/docs

4. Se mudar o banco, delete e recrie com:
   docker-compose down -v
   docker-compose up -d postgres

5. Para debug, use:
   - console.log() no frontend
   - print() no backend
   - docker logs task_manager_db para logs do banco

6. Commit com frequência:
   git add .
   git commit -m "Descrição clara da mudança"
   git push

7. Use branches para features:
   git checkout -b feature/nova-funcionalidade
   git push -u origin feature/nova-funcionalidade
