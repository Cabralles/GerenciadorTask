================================================================================
  COMO FAZER PUSH PARA GITHUB
================================================================================

PASSO 1: Criar repositório no GitHub
  1. Acesse https://github.com/new
  2. Nome do repositório: "task-manager" (ou outro nome de sua preferência)
  3. Descrição: "Gerenciador de Tarefas - React 19 + FastAPI + PostgreSQL"
  4. Selecione: Public (para poder compartilhar)
  5. Clique em "Create repository"
  6. COPIE a URL que aparece (será algo como: https://github.com/seu-usuario/task-manager.git)

PASSO 2: Configurar Git local
  Abra terminal na raiz do projeto:
  
  git config --global user.name "Seu Nome"
  git config --global user.email "seu-email@example.com"

PASSO 3: Inicializar repositório (FAÇA ISSO NA RAIZ DO PROJETO)
  
  cd c:\Users\User\Downloads\Moderntaskmanagerinterface-main\Moderntaskmanagerinterface-main
  
  git init
  git add .
  git commit -m "Initial commit: Task Manager - React 19 + FastAPI + PostgreSQL"

PASSO 4: Adicionar remote e fazer push
  
  git remote add origin https://github.com/SEU-USUARIO/task-manager.git
  
  (substitua SEU-USUARIO pelo seu usuário do GitHub)
  
  git branch -M main
  git push -u origin main

PASSO 5: Verificar no GitHub
  Acesse https://github.com/seu-usuario/task-manager
  Você deve ver todos os arquivos lá!

================================================================================
  TROUBLESHOOTING
================================================================================

❌ Erro: "fatal: not a git repository"
   Solução: Execute "git init" primeiro

❌ Erro: "remote origin already exists"
   Solução: git remote remove origin
           git remote add origin https://github.com/seu-usuario/task-manager.git

❌ Erro: "Authentication failed"
   Solução: 
   - Use token em vez de senha: https://github.com/settings/tokens
   - OU configure SSH: https://docs.github.com/en/authentication/connecting-to-github-with-ssh

❌ Erro: "branch 'main' does not have upstream branch"
   Solução: git push -u origin main
